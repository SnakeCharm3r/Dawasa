Build a **water billing mobile/web application** that allows customers of the Dar es Salaam water utility system to view their water bills and payment status using their **IP number (Account Number)**.

The application must use the following technology stack:

Frontend:
React (Vite or Next.js)

Backend:
Laravel API (RESTful)

Database:
MySQL

External Integration:
DAWASA Billing API

The application must fetch customer billing information from the DAWASA API using the customer's **IP number**.

---

SYSTEM ROLES

1. Customer
   A registered customer who owns the water account.

Login credentials:
Username = IP Number
Password = user generated

Capabilities:

* Login to the system
* View customer profile
* View current balance
* View billing history
* View meter readings
* View payment history
* View control number for bill payment
* Receive bill notifications

---

2. Tenant / Non-Account User

A tenant does not own the account but wants to check the bill status.

Capabilities:

* No login required
* Enter IP number
* Fetch account balance
* View latest bill
* View pending amount
* View control number for payment

Restrictions:

* Cannot view full billing history
* Cannot update account
* Cannot change profile

---

AUTHENTICATION

Customer authentication must use:

Laravel Sanctum or JWT

Login fields:
IP Number
Password

Registration flow:

User enters:
IP Number
Phone Number
Email
Password

System verifies the IP number against the DAWASA API.

If valid:
Create account and allow login.

API STRUCTURE (Laravel)

Endpoints:

POST /api/login

POST /api/register

GET /api/customer/{ip_number}

GET /api/customer/{ip_number}/bills

GET /api/customer/{ip_number}/payments

GET /api/tenant/check/{ip_number}

---

DAWASA API INTEGRATION

Backend must call external API:

GET /dawasa/billing/{ip_number}

Response example:

{
customer_name,
account_number,
meter_number,
current_balance,
control_number,
bills[],
payments[]
}

Backend should cache results to reduce API calls.
